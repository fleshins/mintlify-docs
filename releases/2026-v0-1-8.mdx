This release delivers **more flexible Explorer filters**, **live filter previews**, **better access to related links**, **clearer timestamps across the product**, and **better onboarding**.

Below are the most important changes in this release (not every internal tweak, just the ones we think deliver the most value).

---

## 🔎 **More flexible conversation filtering**

- **Custom Explorer filters:** Teams can now create custom filters in Explorer using detailed account, user, conversation, and message fields — making it easier to narrow results to the exact conversations you want to review.
- **Reusable filter rules:** Custom filters can be named, reused, and added to saved views — so teams can return to important slices of data without rebuilding filters each time.
- **Exclude matching results:** Filter rules can now exclude results as well as include them — giving teams more control when refining large result sets.

## 🛠️ **Smoother filter building**

- **Live preview while building:** The custom filter builder now previews matching conversations as rules are created — helping users see whether a filter will be useful before applying it.
- **More stable counts and previews:** Counts and previews are now more accurate and consistent when editing or clearing draft filters — so teams can trust the feedback they see while refining filters.
- **Save rules before matches exist:** Custom rules can now be completed and saved even when there are no current matches — which helps teams prepare saved views in advance for future activity.

## 🔗 **Better access to related records**

- **Clickable links in details:** Links in conversation and activity details are now clickable and open in a new tab — making it faster to jump from Brixo to the related external page.
- **Clearer link visibility:** Important links are surfaced more clearly in record details, including on existing records when link data was already available — making older history easier to use and investigate.

## 📅 **Clearer dates across the product**

- **Year included in timestamps:** Dates and times across the product now include the year — reducing ambiguity when reviewing older conversations and activity.

Thanks for using Brixo! We hope these updates make your day-to-day workflows smoother, faster, and more intuitive.

## 🚀 **Faster, clearer onboarding**

- **Clearer setup guidance:** The product documentation step now uses simpler wording and a more polished setup flow — helping new users understand what Brixo is doing and move through setup with less confusion.
- **More transparent scan progress:** Documentation scanning now gives a clearer sense of current progress and what happens next — so teams have better visibility during setup instead of wondering whether the process is working.

## 🧭 **Simpler review and recovery**

- **Easier documentation review:** It is now more straightforward to review the documentation Brixo finds, confirm results, rescan another site, or add missing content — making it easier to fix issues and complete setup without extra back-and-forth.
- **Clearer recovery guidance:** When a scan does not complete or misses expected content, the product now provides clearer guidance on next steps — helping teams recover faster instead of getting stuck during setup.

## ✅ **More dependable setup outcomes**

- **More reliable setup processing:** Setup and automatic organization are now more dependable, with fewer duplicate processing issues and fewer stuck workflows — making successful setup more likely on the first try.
