This release delivers **background imports so you can start sooner**, **import progress that stays visible across the app**, **more complete documentation discovery (including docs/support subdomains)**, and **clearer onboarding and mapping states**.

Below are the most important changes in this release (not every internal tweak, just the ones we think deliver the most value).

---

## ⏱️ **Faster setup with progress you can follow**

- **Start using the app while imports run:** You can move into the app sooner while your import continues in the background — You don’t have to wait on a long import before getting started.
- **Import progress stays visible:** Import progress remains visible across the app instead of requiring you to stay on one screen — It’s easier to keep track of long-running uploads while doing other work.
- **Email notification when an import is ready:** You’ll receive an email when an import completes and is ready — You can step away and return at the right time without repeatedly checking.

## 📚 **More complete and dependable documentation imports**

- **Improved content discovery across docs domains:** Documentation imports are better at finding useful content, including pages on support or docs subdomains — You’re more likely to capture the right pages without manual workarounds.
- **Better handling for large documentation sites:** Scans process larger documentation sites more effectively to bring in more complete source material — You get broader coverage when importing sizable docs portals.
- **Clearer outcomes when a site can’t be reached:** If a documentation site can’t be reached, the app fails faster instead of appearing stuck — You get feedback sooner and can fix the issue or try a different source.
- **More reliable retries after failed scans:** Retry behavior is more reliable to reduce irrelevant or partial results after a failure — You get cleaner import results with fewer confusing leftovers from earlier attempts.

## 🧭 **A clearer onboarding and import flow**

- **More guided transition after selecting content:** The onboarding flow more clearly shows that your selected content is being reviewed — The process feels easier to follow and less abrupt.
- **Clearer import progress messaging:** Import progress text is clearer and more accurate, with less confusing wording and fewer misleading estimates — You can better understand what’s happening and what to expect next.
- **More understandable loading and refresh actions:** Loading states during import mapping are easier to interpret, and refresh actions are labeled more clearly — It’s easier to recover from delays and know what each action will do.

## ✨ **Everyday usability polish**

- **Close modals and side panels with Esc:** Modals and side panels can now be closed using the Esc key — Navigation is faster and requires fewer clicks.
- **Invite links copy automatically:** Creating a team invite link now copies it automatically and confirms it was copied — Sharing invite links is quicker and less error-prone.
- **Smoother teammate invites by email:** Email invites are more forgiving, including when clicking away from the input field — You’re less likely to lose what you typed or need to re-enter information.
- **Easier-to-use filters and cleaner auth/profile screens:** Filters are easier to understand when including or excluding results, sign-in and sign-up screens have a cleaner layout, and profile details show email information more clearly when available — Everyday navigation feels clearer and key account details are easier to find.

Thanks for using Brixo! We hope these updates make your day-to-day workflows smoother, faster, and more intuitive.
